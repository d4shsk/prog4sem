# Лабораторная работа 2. Паттерны проектирования. Декоратор

В рамках данной лабораторной работы реализован код с использованием классического паттерна проектирования «Декоратор», который:
- возвращает данные в форматах JSON, YAML и CSV;
- использует строгий интерфейс (на базе абстрактного класса `abc.ABC` и декоратора `@abstractmethod`).

## Структура проекта

Проект разделен на два основных файла для логического разделения основного кода и тестов:

* `data_providers.py` — содержит интерфейс `DataProvider`, базовый компонент `SimpleDataProvider` (возвращающий сырые данные) и набор декораторов:
  * `JSONDecorator` — форматирует данные в строку JSON.
  * `YAMLDecorator` — форматирует данные в строку YAML.
  * `CSVDecorator` — форматирует данные в строку CSV с выгрузкой заголовков.
* `test_decorators.py` — содержит набор модульных тестов для проверки работоспособности компонентов.

## Описание Unit-тестов

Для проверки надежности кода используется встроенный фреймворк `unittest`. В классе `TestDataDecorators` реализованы проверки следующих сценариев:

1. Тестирование базового компонента (`test_base_provider`): Проверяет, что `SimpleDataProvider` корректно хранит и возвращает переданный ему исходный список словарей без изменений.
2. Тестирование JSON-форматирования (`test_json_decorator`): Проверяет работу `JSONDecorator`.
3. Тестирование YAML-форматирования (`test_yaml_decorator`): Проверяет работы `YAMLDecorator`.
4. Тестирование CSV-форматирования (`test_csv_decorator`): Проверяет работу `CSVDecorator`. 
5. Тестирование краевых случаев (`test_empty_csv`): Проверяет поведение CSV-декоратора при передаче пустого списка данных. Удостоверяется, что код не вызывает ошибок и возвращает пустую строку.
