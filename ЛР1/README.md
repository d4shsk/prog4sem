# Лабораторная работа 1. Ряд Фибоначчи с помощью итераторов

## Задание 1. Реализация итераторов

Были реализованы два механизма генерации чисел Фибоначчи:

1. Упрощенный итератор (через `__getitem__`): Реализован подход, при котором объект становится итерируемым за счет поддержки индексации. Внутри класса используется кэширование: при запросе индекса список чисел динамически достраивается до нужного размера. Завершение цикла итерации происходит при выбросе исключения `IndexError`, когда достигается заданный лимит.
2. Классический итератор (через `__iter__` и `__next__`): Реализован стандартный протокол итератора. Класс хранит текущее состояние (шаг и два последних вычисленных числа) в памяти, не сохраняя весь ряд целиком, что делает его более эффективным по потреблению ресурсов. Остановка генерации контролируется исключением `StopIteration`.

## Задание 2. Изучение и реализация сопрограмм (Coroutines)

* Теоретический вывод: Главное отличие сопрограммы от обычного генератора заключается в двунаправленности работы. Если классический генератор только *производит* значения и приостанавливает работу (`yield`), то сопрограмма способна еще и *принимать* данные извне в точку остановки с помощью метода `.send(value)`.
* Практическая реализация: Написана сопрограмма, которая бесконечно выдает числа Фибоначчи. В нее внедрен механизм управления состоянием: если во время работы передать в нее значение `True` через метод `send(True)`, сопрограмма перехватывает этот сигнал и сбрасывает свои внутренние переменные, начиная отсчет последовательности заново (с 0 и 1).
